TASK 1 — Pin Top 3 Skills:  Forward Deployed Engineering /
                            Applied AI / Agent Orchestration
================================================================

LinkedIn DOES still allow manual drag-and-drop reorder.  The
top 3 positions act as the "pinned" Top Skills (no separate
pin toggle — position 1-3 IS the pin).

Steps:

  1. Go to your profile page.
     URL:  https://www.linkedin.com/in/dbhavery/

  2. Scroll down to the "Skills" section.

  3. Click the PENCIL icon at the TOP-RIGHT of the section card.
     (Not on individual skill rows — at the section header level.)

  4. In the edit panel, click the THREE-DOT overflow at the
     top-right, then click the up-down "Reorder" icon.

  5. Hover over a skill row.  A HAMBURGER drag handle appears
     on the LEFT side of the row.

  6. Drag-and-drop into this order:

        1.  Forward Deployed Engineering
        2.  Applied AI
        3.  Agent Orchestration

     The first 3 render as "Top Skills" on your profile and on
     the recruiter search-result card.

  7. Click X to exit edit mode.  Changes save on drop.

Browser tip:
  Chrome is the most reliable.  Safari has had drag-and-drop
  bugs through 2025.  Use Chrome on desktop, not the mobile app
  (the mobile drag handle is finicky).

TASK 4 — Fix Companion case-study link preview on /aether
        (Next.js per-route Open Graph metadata)
================================================================

WHY:  When LinkedIn fetches https://dbhavery.dev/aether for a
link preview, it currently falls back to dbhavery.dev root meta.
That's why the "Add a link" feature pinned dbhavery.dev twice
instead of differentiating /aether.  Fix needs to happen on the
portfolio side.

This belongs to the OTHER Claude Code agent working on the
portfolio.  Pass them this section.

CANONICAL PATTERN (Next.js 15 App Router):

  Add to file:  app/aether/page.tsx

  ```tsx
  import type { Metadata } from 'next';

  export const metadata: Metadata = {
    title: 'Companion — Don Havery',
    description: 'Voice-agent harness with 7 sub-agents and '
               + 'cross-provider behavioral parity at 449ms.',
    openGraph: {
      title: 'Companion — Voice-Agent Harness',
      description: '7 coordinated sub-agents · 5-model routing · '
                 + '449ms end-to-end voice latency.',
      url: 'https://dbhavery.dev/aether',
      images: [{
        url: '/og/companion.png',
        width: 1200,
        height: 630,
        alt: 'Companion case study',
      }],
      type: 'website',
    },
  };
  ```

ROOT LAYOUT also needs:

  Add to file:  app/layout.tsx

  ```tsx
  export const metadata: Metadata = {
    metadataBase: new URL('https://dbhavery.dev'),
    // ... rest of the metadata
  };
  ```

  Without metadataBase, LinkedIn pulls relative image paths
  (/og/companion.png) against localhost during build → shows
  nothing.

GOTCHA:  generateMetadata is silently dropped if the page uses
'use client' or is wrapped in a client-only layout.  Verify
server-rendering with:

    curl -A "LinkedInBot/1.0" https://dbhavery.dev/aether | grep og:

You should see og:title, og:description, og:image lines pointing
at the Companion-specific values.

AFTER DEPLOY:  force LinkedIn to re-scrape using the official
post inspector:

    https://www.linkedin.com/post-inspector/

Paste https://dbhavery.dev/aether and click Inspect.  This
flushes LinkedIn's cache so the next time anyone shares /aether,
the correct preview lands.

Per-route metadata should also be added for the other showcase
routes (/cige → Darkroom, /herald → Wire, /vault → Albums, etc.)
so each gets its own LinkedIn preview.
